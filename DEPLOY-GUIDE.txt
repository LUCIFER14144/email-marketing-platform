OPTION A: Using Render.com (EASIEST - No Git Required!)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Step 1: Upload to GitHub Manually
──────────────────────────────────
1. Open browser → https://github.com
2. Sign up (if new) or login
3. Click [+] → "New repository"
4. Name: email-marketing-platform
5. Click "Create repository"
6. Click "uploading an existing file"
7. Drag ALL files from:
   C:\Users\Eliza\Desktop\webemailsending
   
   ⚠️ SKIP THESE FOLDERS:
   - node_modules (too large)
   - uploads (user data)
   
   ⚠️ SKIP THIS FILE:
   - .env (contains passwords)

8. Click "Commit changes"

✅ GitHub upload complete!

Step 2: Deploy to Render
─────────────────────────
1. Open new tab → https://render.com
2. Click "Get Started for Free"
3. Sign up with GitHub account
4. Click "New +" → "Web Service"
5. Click "Connect GitHub"
6. Find "email-marketing-platform"
7. Click "Connect"

8. Fill in the form:
   ┌─────────────────────────────────────┐
   │ Name: my-email-platform             │
   │ Environment: Node                   │
   │ Build Command: npm install          │
   │ Start Command: npm start            │
   │ Instance Type: Free                 │
   └─────────────────────────────────────┘

9. Scroll down to "Environment Variables"
10. Click "Add Environment Variable"
11. Add:
    Key: SESSION_SECRET
    Value: make-this-a-very-long-random-string-abc123xyz789

12. (Optional) Add Gmail credentials:
    Key: GMAIL_SERVICE  Value: gmail
    Key: GMAIL_USER     Value: your-email@gmail.com
    Key: GMAIL_PASSWORD Value: your-app-password

13. Click "Create Web Service"

14. ⏳ Wait 5-10 minutes (watch the logs)

15. ✅ DONE! Your app is LIVE!

Your URL: https://my-email-platform.onrender.com

TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

❌ "Cannot find module..."
   → Check all files were uploaded to GitHub
   → Verify package.json is included

❌ "Application failed to start"
   → Check Render logs
   → Verify environment variables are set
   → Make sure SESSION_SECRET is added

❌ "Page not loading"
   → Wait 10 minutes for first deployment
   → Check deployment status in Render dashboard
   → Look for errors in logs

❌ Email not sending
   → Add SMTP credentials as environment variables
   → Use Gmail app password (not regular password)
   → Check SMTP settings


KEEPING YOUR APP AWAKE (Free Tier)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Free Render apps "sleep" after 15 min of inactivity.
First request after sleep takes ~30 seconds.

To keep it awake:
1. Go to https://uptimerobot.com (free)
2. Add new monitor
3. Type: HTTP(s)
4. URL: your Render URL
5. Interval: 14 minutes

This pings your app every 14 min to keep it awake!
